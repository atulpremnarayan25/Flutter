App Name: Eventify – Smart Event Reminder App
________________


1️⃣ Project Overview
🎯 Objective
Build a fully functional Flutter Event Reminder App that allows users to:
* Create events

* Schedule reminders

* Receive local notifications

* Manage upcoming and past events

* Store data locally

* Maintain clean UI/UX

* Follow secure coding practices

The app must demonstrate:
   * Multi-screen navigation

   * Proper state management

   * Local database usage

   * Form validation

   * Notifications

   * Clean architecture

________________

4️⃣ State Management
Use:
✅ Provider (Recommended)
Structure:
                                       * EventProvider

                                       * AuthProvider

Responsibilities:
EventProvider:
                                          * Add event

                                          * Delete event

                                          * Update event

                                          * Fetch events

                                          * Mark completed

AuthProvider:
                                             * Login

                                             * Register

                                             * Logout

                                             * Check authentication state

________________


5️⃣ Database Design
📁 Use Hive (Local NoSQL DB)
Create Event Model:
Fields:
id (String – UUID)
title (String)
description (String)
dateTime (DateTime)
isCompleted (bool)
createdAt (DateTime)
Security rules:
                                                * No sensitive data stored in Hive

                                                * Passwords must NOT be stored in Hive

                                                * Only store event data

________________


6️⃣ Notification System
Use:
flutter_local_notifications
timezone
Requirements:
                                                   * Schedule notification when event is saved

                                                   * Cancel notification if event deleted

                                                   * Unique notification ID per event

                                                   * Handle app in foreground and background

                                                   * Ask notification permission properly (Android 13+)

9️⃣ Validation Rules
📝 Form Validation
Event Form:
                                                            * Title cannot be empty

                                                            * Date cannot be past

                                                            * Time must be valid

Auth Form:
                                                               * Email regex validation

                                                               * Password >= 6 characters

Display errors using:
                                                                  * TextFormField validator

                                                                  * SnackBar for general errors

________________


🔟 Error Handling
                                                                     * Try/catch around all database calls

                                                                     * Try/catch around notification scheduling

                                                                     * Show user-friendly error messages

                                                                     * No raw exception printing in UI

                                                                     * Use debugPrint instead of print
